❓ Frequently Asked Questions

What is Markdown and why should I use it?

Markdown is a lightweight markup language created by John Gruber in 2004. It uses simple symbols like # for headings, ** for bold, and - for lists. It's used across GitHub, documentation sites, blogs, and note-taking apps. Unlike HTML, Markdown is human-readable even in raw form.

What is GitHub Flavored Markdown (GFM)?

GFM extends standard Markdown with: tables (using | pipes), task lists (- [x] done), strikethrough (~~text~~), fenced code blocks with language hints (```js), and autolinked URLs. This editor supports all GFM features.

How do I create a table in Markdown?

Use pipes and hyphens:

| Header 1 | Header 2 |
|----------|----------|
| Cell 1   | Cell 2   |

Align columns with colons: |:---| left, |:---:| center, |---:| right.

How do I convert Markdown to HTML programmatically?

Popular libraries:

  • JavaScript: marked.parse('# Hello') (marked.js)
  • Node.js: markdown-it or remark
  • Python: import markdown; markdown.markdown('# Hello')
  • CLI: pandoc input.md -o output.html
Is my data safe in this editor?

Yes. This editor runs 100% in your browser — no data is sent to any server. There's no account, no cloud sync, and no tracking of your content. Your Markdown stays on your machine. Check the network tab in DevTools to verify: zero requests.

Can I share my Markdown document with someone?

Click the 🔗 Share button in the header. It generates a URL with your Markdown content encoded in the URL parameter. Anyone who opens the link sees your exact document loaded and rendered instantly. You can also download the .md file directly.
