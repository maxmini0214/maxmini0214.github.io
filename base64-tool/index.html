❓ Frequently Asked Questions

How do I convert an image to Base64?▼
Click the "📁 File → Base64" tab, then drag and drop your image. The tool instantly converts it to a Base64 string with a data: URI prefix ready for HTML/CSS use.
What is URL-safe Base64?▼
URL-safe Base64 (RFC 4648 §5) replaces + with -, / with _, and removes trailing = padding. This makes encoded strings safe for URLs, query parameters, and filenames without additional percent-encoding.
Can I preview a Base64 image?▼
Yes! Use the "🖼️ Base64 → Preview" tab. Paste your Base64 string (with or without the data: prefix) and click Preview. Supports JPEG, PNG, GIF, WebP, SVG.
Is my data sent to any server?▼
No. All processing happens 100% in your browser using the JavaScript btoa/atob and FileReader APIs. You can verify by using the tool with your network tab open — zero requests are made.
What's the max file size?▼
The tool handles files up to ~50MB depending on your browser's memory. For best performance, keep files under 10MB. Large files are processed locally using the FileReader API.
What is a Data URI?▼
A Data URI (data:[mime];base64,[data]) embeds file content directly into HTML/CSS. It eliminates an HTTP request — ideal for small images, icons, and fonts in web development.
How do I decode Base64 back to a file?▼
For images, use the "🖼️ Base64 → Preview" tab and click "💾 Download File". For text, use the Text tab's Decode button. The tool auto-detects MIME types from data: URIs.
What's the difference between standard and URL-safe Base64?▼
Standard Base64 uses A-Z, a-z, 0-9, +, / and = padding. URL-safe Base64 replaces + with -, / with _, and optionally strips = padding — making it safe for URLs without percent-encoding. Use it for JWTs, API tokens, and URL query params.
