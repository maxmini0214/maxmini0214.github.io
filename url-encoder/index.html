❓ Frequently Asked Questions

What is the difference between encodeURI and encodeURIComponent?

encodeURI encodes a full URI but preserves special characters like :, /, ?, and #. encodeURIComponent encodes everything except letters, digits, and - _ . ~, making it ideal for encoding individual query parameter values.

How do I build a query string from parameters?

Click the "🔗 Query String Builder" tab. Enter key-value pairs and optionally a base URL. The tool generates a properly encoded query string. You can also paste an existing URL and click "Parse" to extract its parameters.

Can I encode multiple URLs at once?

Yes! Click the "📦 Batch" tab. Paste multiple URLs or strings, one per line, then click "Encode All" or "Decode All". Each line is processed independently, maintaining the same order.

Is this tool safe for sensitive data?

Absolutely. This tool runs 100% in your browser — no data is sent to any server. Verify by checking the Network tab in DevTools: zero outgoing requests. See also: Base64 Encoder and Hash Generator.

How do I encode a URL with spaces?

Spaces are encoded as %20 with both encodeURIComponent and encodeURI. Simply paste your text and click Encode — the tool handles all special characters automatically.

Can I decode a double-encoded URL?

Yes! If a URL was double-encoded (e.g., %2520 instead of %20), click Decode multiple times. Each click removes one layer of encoding until the text is fully readable.

What is percent encoding?

Percent encoding (URL encoding) replaces unsafe characters with a % followed by two hexadecimal digits. For example: space → %20, ampersand → %26. Check the "📖 Encoding Reference" tab for the complete table.

What characters does encodeURIComponent NOT encode?

encodeURIComponent preserves A-Z, a-z, 0-9, and the characters - _ . ~. Everything else gets percent-encoded. Use JWT Decoder to inspect encoded JWT tokens.
