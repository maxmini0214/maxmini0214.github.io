❓ Frequently Asked Questions

What is a QR code and how does it work?
A QR (Quick Response) code is a two-dimensional barcode that stores data in a grid of black and white squares. When scanned by a camera or QR reader app, the pattern is decoded back into the original text, URL, or data. QR codes use Reed-Solomon error correction, meaning they can still be read even if up to 30% of the code is damaged or obscured.
How do I create a QR code for WiFi?
Use this format in the text field:

WIFI:T:WPA;S:YourNetworkName;P:YourPassword;;

Replace WPA with WEP or nopass as needed. When scanned, most phones will offer to connect to the WiFi network automatically without typing the password. Great for guest networks, cafes, and events.
Is there a character limit for QR codes?
QR codes can store up to 4,296 alphanumeric characters or 2,953 bytes of binary data. However, more data = denser code = harder to scan. For best results, keep content under 300 characters. For long URLs, use a URL shortener first.
How do I generate a QR code in JavaScript?
Using the qrcode npm package:

const QRCode = require('qrcode');
QRCode.toDataURL('Hello World', (err, url) => {
  console.log(url); // data:image/png;base64,...
});

In the browser with canvas:
QRCode.toCanvas(canvas, 'text', { width: 256 });

This tool uses the same qrcode library under the hood.
Can QR codes be customized with colors or logos?
Standard QR codes are black and white, but the spec allows color changes as long as contrast remains high. Adding a small logo in the center works because of Reed-Solomon error correction — the code can tolerate ~30% damage. This tool generates standard black-on-white codes for maximum compatibility across all scanners.
