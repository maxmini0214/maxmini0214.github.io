❓ Frequently Asked Questions

How does an online diff checker work?

A diff checker uses the Longest Common Subsequence (LCS) algorithm to find the minimal set of changes between two texts. Lines are classified as added (green), deleted (red), or modified (yellow). Example in JavaScript:

// Simple LCS-based diff
function diff(a, b) {
  const dp = Array.from({length: a.length+1},
    () => new Array(b.length+1).fill(0));
  for (let i=1; i<=a.length; i++)
    for (let j=1; j<=b.length; j++)
      dp[i][j] = a[i-1]===b[j-1]
        ? dp[i-1][j-1]+1
        : Math.max(dp[i-1][j], dp[i][j-1]);
  return dp;
}

How do I compare two files from the command line?

On Linux/macOS use diff file1.txt file2.txt or diff -u file1.txt file2.txt for unified format. For Git repositories, git diff shows staged changes. On Windows, use fc file1.txt file2.txt or install a tool like WinMerge.

What's the difference between diff, patch, and merge?

Diff shows differences. Patch is a file containing those differences that can be applied: patch < changes.patch. Merge combines changes from two branches, resolving conflicts where both sides edited the same lines.

Can I compare JSON, YAML, or code files?

Yes. This tool compares any plain text line-by-line. For best results with JSON, format both versions first (try our JSON Formatter), then paste. For minified code, pretty-print before comparing.
