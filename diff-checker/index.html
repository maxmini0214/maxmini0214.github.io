❓ Frequently Asked Questions

How does an online diff checker work?

A diff checker uses the Longest Common Subsequence (LCS) algorithm to find the minimal set of changes between two texts. Lines are classified as added (green), deleted (red), or modified (yellow). This tool goes further with word-level diffing inside modified lines. Example:

// LCS-based diff
function diff(a, b) {
  const dp = Array.from({length: a.length+1},
    () => new Array(b.length+1).fill(0));
  for (let i=1; i<=a.length; i++)
    for (let j=1; j<=b.length; j++)
      dp[i][j] = a[i-1]===b[j-1]
        ? dp[i-1][j-1]+1
        : Math.max(dp[i-1][j], dp[i][j-1]);
  return dp;
}
What is word-level diff highlighting?

Word-level diff shows exactly which words changed within a modified line, not just that the line changed. If you changed Hello World to Hello Earth, word-level diff highlights only World → Earth instead of marking the entire line as modified. This makes code reviews and config comparisons much faster.

How do I compare two files using drag and drop?

Drag any text file directly onto the Original or Modified text area. Supported formats: .txt, .json, .csv, .yaml, .yml, .md, .js, .ts, .py, .go, .rs, .java, .html, .css, .xml, .sql, .env, .toml, and more. Max 10MB per file.

What does "Ignore Whitespace" do?

When enabled, lines are compared after trimming leading/trailing whitespace and collapsing multiple spaces into one. This is useful for comparing code where indentation style changed (tabs vs spaces, 2 vs 4 spaces) but the logic stayed the same. Toggle it to focus on meaningful changes.

What's the difference between Split and Unified view?

Split view shows original and modified text side-by-side in two columns — great for visual comparison. Unified view interleaves all changes in a single column (like git diff output), which is more compact and easier to read sequentially. Switch between them anytime without re-comparing.

Is my data safe when using this tool?

Yes — 100% client-side. No data is sent to any server. Open DevTools → Network tab to verify: zero requests on compare. History is stored in your browser's localStorage only. This tool even works offline after the page loads.

Can I compare JSON, YAML, or code files?

Yes. This tool compares any plain text with word-level precision. For JSON, format both versions first (try our JSON Formatter), then paste or drag files. The diff shows exact line and word changes. Works with any language: JavaScript, Python, Go, Rust, YAML, CSV, SQL, and more.

How do I share a comparison with my team?

Click the 🔗 Share button after comparing. It encodes both texts into a URL that anyone can open to see the exact same diff. No server storage — everything is in the URL itself. For very large texts, use 📋 Copy Diff and paste into your team chat.

How does comparison history work?

Every comparison is automatically saved to your browser's localStorage. Click the 📜 History button to see your last 8 comparisons with timestamps and preview. Click any entry to restore it instantly. History never leaves your browser and persists between sessions.

How is this different from git diff?

git diff works on files in a Git repository from the command line. This tool works on any text in your browser — no Git needed. It adds word-level highlighting, ignore whitespace toggle, split/unified views, and shareable URLs. For quick comparisons of snippets, config changes, or API responses, a browser tool is faster than setting up files.

What's the difference between diff, patch, and merge?

Diff shows differences. Patch is a file containing those differences that can be applied: patch < changes.patch. Merge combines changes from two branches, resolving conflicts where both sides changed the same lines.
