Frequently Asked Questions

Is this JWT decoder safe to use with production tokens?
Yes. This decoder runs 100% in your browser. No data is ever sent to any server. Your tokens never leave your machine. You can verify this by opening DevTools → Network tab — zero requests are made when you paste a token.
Can this tool verify JWT signatures?
This tool decodes and inspects JWT structure, header, payload, and claims. It does not verify cryptographic signatures since that requires the secret key or public key, which should never be shared with a web tool. For signature verification, use a local library like jsonwebtoken (Node.js) or PyJWT (Python).
What JWT algorithms are supported?
The decoder works with any JWT regardless of algorithm — HS256, RS256, ES256, PS256, EdDSA, and more. Since it decodes the base64 structure without needing the signing key, the algorithm doesn't matter for inspection.
How do I check if a JWT token is expired?
Paste your JWT and look at the Decoded Claims section below the panels. The exp (Expiration) claim shows the expiry date with color coding — green means valid, red means expired. You can also see iat (Issued At) and nbf (Not Before) timestamps.
Can I share a decoded JWT with my team?
Yes! Click the 🔗 Share button to generate a URL with your JWT encoded in the URL parameters. Anyone opening the link will see the same decoded result instantly — great for debugging auth issues together.
What is the difference between JWT header and payload?
The header (JOSE Header) contains metadata: the signing algorithm (alg) and token type (typ). The payload contains the actual claims — user data, permissions, expiration time, issuer, and any custom fields your application needs.
