❓ Frequently Asked Questions

What is the Pomodoro Technique?
The Pomodoro Technique is a time management method created by Francesco Cirillo in the late 1980s. It uses a kitchen timer (pomodoro = tomato in Italian) to break work into 25-minute focused intervals called "pomodoros", separated by 5-minute breaks. After 4 pomodoros, take a longer 15–30 minute break. Studies show this reduces mental fatigue and improves sustained concentration.
Why 25 minutes? Can I change the duration?
25 minutes is the classic Pomodoro interval — long enough for meaningful work, short enough to maintain focus. Many people customize it: developers often use 50/10, students prefer 25/5, and writers sometimes use 45/15. This tool offers Work (25m), Short Break (5m), and Long Break (15m) modes.
How do I set a Pomodoro timer in code?
JavaScript:
setInterval(() => { if (--s <= 0) { clearInterval(t); alert('Done!'); }}, 1000);

Python:
import time; time.sleep(25*60); print('Pomodoro done!')

Terminal (macOS/Linux):
sleep 1500 && echo "Done" && afplay /System/Library/Sounds/Glass.aiff

Windows:
timeout /t 1500 && echo Done
Does this timer work offline?
Yes! This Pomodoro timer runs entirely in your browser with JavaScript. Once the page loads, no internet connection is needed. Session count is tracked in memory during your browser session. The timer uses the Web Audio API for sound alerts and the Notification API for desktop notifications.
How can I share my timer settings with a team?
Click the 🔗 Share Settings button to generate a URL that encodes your current timer mode. Send this link to teammates — they'll open the timer with the exact same configuration. The URL uses parameter encoding — no data is sent to any server.
